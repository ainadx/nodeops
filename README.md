# NodeOps — Smart Traffic Node Companion (AiNa build)

A complete product package generated by the AiNa pipeline: a **document set**
(one markdown doc per stage) alongside a **working codebase** — the NodeOps
operator console.

NodeOps is the single companion for the whole life of a Smart Traffic Node
(solar + LTE open-hardware signal controller): **provision → bring up → operate →
monitor → maintain**. This package delivers the reference Cloud↔App operator
console + mock cloud/device seams that the production mobile clients and node
firmware integrate against.

## Run it
```bash
cd app
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000      # open http://localhost:8000
pip install pytest httpx && pytest -q           # 13 passed
```

## Stack
FastAPI + Jinja2/HTMX · server-rendered design system · SQLite/in-memory
(reference) → Postgres + Timescale + MQTT (production) · Docker on a VPS.
Chosen by the AiNa classifier (backend-heavy, internal-tool).

## Safety
Signal control is confirmed, time-bounded, logged, refused without a live link,
and fails safe on the node. Health/signal indicators use letter + shape cues,
not color alone. See `docs/05-security.md`.
